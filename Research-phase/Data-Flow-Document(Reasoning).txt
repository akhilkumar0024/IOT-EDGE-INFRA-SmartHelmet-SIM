Sad Path 1 — Bluetooth pairing fails
The helmet powers on, tries to pair with the smartphone, and gets no response or a failed pairing.
What should happen? 

What does the rider see?
Rider should see the helmet is not connected to the smartphone, and the telemetry systems are not operational, and the alert system is not operational
It should display what the helmet is trying to do, like trying to connect to the smartphone, and it should also display the number of retries left 
in that poweron event

Does the helmet keep retrying or give up immediately?
It should not give up immediately, instead it should have atleast 5 tries for that poweron

Is there a maximum number of retries before it stops?
The maximum number of tries should be 5 per poweron

After it stops retrying — what state is the helmet in?
If the helmet never manages to connect to the smartphone even after 5 retries it should now display the message to visit the maintainance shop, 
one thing it should never do i clear the buffer memory on its own under not circumstancer until the confirmation is revieved from the smartphone 
that the data is successfully uploaded in the cloud, this is to ensure that the data is not lost
and it should keep track of each retry in memory if the cumulative retry ever croses a number like(50) in all the powerons it suggest a maintainance 
issue and shoukld advice the user to take helmet for maintainance
