Conflict 2 — Who triggers the 30-second alert window
Edge Processing (Helmet) sends crash event → but the 30-second countdown and cancel mechanism involves both Helmet HUD and Smartphone app simultaneously.
The conflict: if Smartphone is dead, can the rider cancel from the Helmet alone? Currently no actor owns this answer. Both Helmet and Smartphone actors are silent on what happens when the other is unavailable during the countdown.

Answer : 
The flow is helmet mounted sensors determine a crash make a request to the smartphone to send the  alert. 
the smartphone will then send a request to the telemetry infrastructure, the processing infra will act to determine if the alert is real or a false positive 
if the smartphone is dead, the alerts or any telemetry data will not be sent to the telemetry infrastructure. 
but the cancellation of alert can be done from any of the two, helmet or smartphone. 
thats why i said smartphone is the single point point of failure between the helmet and the telemetry infrastructure.
but having backup comms in the helmet will add to the cost and complexity of the project and such redundancy is not required for now
maybe in an actual hardware implementation, but of as an next update certainly not in this simulation
